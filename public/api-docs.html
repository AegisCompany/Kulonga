API Reference

Documentação oficial para integração com o ecossistema Kulonga. Siga os padrões RESTful para interagir com dados escolares de forma segura.

Autenticação e Segurança

O Kulonga utiliza autenticação baseada em **API Keys** para sistemas externos e **JWT Tokens** para aplicações de utilizador. Todas as requisições externas devem incluir o cabeçalho X-API-KEY.

Exemplo de Requisição (cURL)

curl -X GET https://api.kulonga.com/api/academic/students \
  -H "X-API-KEY: KUL-VOSSA-CHAVE-AQUI"

1. Autenticação e Segurança (/api/auth)

Essenciais para permitir que outros sistemas validem utilizadores do Kulonga.

POST /api/auth/login

Autenticação de utilizadores e geração de tokens JWT.

GET /api/auth/me

Obter detalhes do perfil do utilizador autenticado.

GET /api/auth/oauth/google

Iniciar fluxo de autenticação via Google.

POST /api/auth/mfa/setup

Configuração de autenticação de dois fatores.

2. Gestão Académica (/api/academic)

Permite a sincronização de dados escolares com outras plataformas de ensino.

GET /api/academic/students

Listar todos os alunos da instituição (com filtros por turma).

POST /api/academic/students

Matricular novos alunos via sistema externo.

GET /api/academic/subjects

Listar disciplinas e currículos.

GET /api/academic/grades

Consultar notas e aproveitamento escolar.

POST /api/academic/grades

Lançamento de notas (útil para integração com apps de professores).

GET /api/academic/schedules

Consultar horários de aulas e exames.

3. Gestão de Professores (/api/teachers)

Para sistemas de RH ou gestão de equipas.

GET /api/teachers/

Listar corpo docente e suas especializações.

POST /api/teachers/vacations

Solicitação de férias ou ausências.

GET /api/teachers/vacations

Consultar estado de pedidos de ausência.

4. Financeiro e Pagamentos (/api/financial)

Crucial para integração com bancos, caixas ou sistemas de faturação.

GET /api/financial/invoices

Listar faturas e estados de pagamento (propinas, taxas).

GET /api/financial/summary

Obter resumo financeiro (total pago vs pendente).

POST /api/financial/payments/callback

Endpoint para receber confirmações de pagamento de gateways externas (ex: EMIS/Multicaixa).

5. Certificação e Documentos (/api/certification)

Para validação de documentos oficiais.

GET /api/certification/verify/:code

Validar a autenticidade de um certificado ou declaração via código QR.

GET /api/certification/templates

Listar modelos de documentos disponíveis.

6. Notificações (/api/notifications)

Para envio de alertas multicanal.

POST /api/notifications/send

Enviar notificações push ou SMS para encarregados de educação e alunos.

Webhooks

Receba notificações em tempo real sobre eventos na sua instituição.

POST /api/external/webhooks

Registar um novo endpoint de webhook.

GET /api/external/webhooks

Listar webhooks ativos.
